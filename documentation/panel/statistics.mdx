Statistics shows what is happening across users, nodes, and subscriptions.

## Realtime Data

Use realtime data to check the current state:

- total traffic
- online users
- active, limited, disabled, expired, and on-hold users
- node traffic distribution
- server CPU, RAM, disk, uptime, and core count

If realtime charts are empty, wait for traffic to start or check whether the selected node is connected.

## Historical Data

Historical data helps you review a selected time range. Choose a range before reading the charts.

Use historical data when you want to answer questions such as:

- which node used the most traffic
- how much traffic was used during a period
- whether user counts changed after a bulk operation
- whether subscription updates increased after a client change

## Admin And Client Filters

Some charts can be filtered by admin. This helps owners compare usage between admins or teams.

Subscription statistics show which client apps or agents are updating subscription links. Use this when users report subscription problems from one specific app.

## When Numbers Look Wrong

- Check that the node is connected.
- Confirm the selected time range.
- Remember that status changes can affect history.
- If usage was reset, chart history may be cleared depending on cleanup settings.
